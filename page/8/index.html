Session 使用 SQLServer

建置 Database

使用 aspnet_regsql.ext 工具建立

複製路徑 C:\Windows\Microsoft.NET\Framework64\v4.0.30319
命令提示字元輸入


可看到資料庫已建置完成

修改 Web.Config

Web.Config 裡的 system.web 區塊 seesionState 修改

<sessionState mode="SQLServer" allowCustomSqlDatabase="true" sqlConnectionString="data source=<serverIP>;initial catalog=ASPState;user id=<userName>;password=<strongPassword>"  sqlCommandTimeout="60" cookieless="false" cookieName="ASP.NET_SessionId" timeout="60"/>

NLB 網路負載平衡

發表於 2019-04-26 | 分類於 MIS

網路負載平衡設定

IP 皆為固定，叢集 IP 不能被使用
  1. A server IP ： 172.27.59.161
  2. B server IP ： 172.27.59.162
  3. 叢集 IP ： 172.27.59.163

安裝 NLB

Server 設定

  1. Server Manager → Add roles and features

  2. Next

  3. Next

  4. Next

  5. 勾選 Network Load Balancing → Next

  6. 勾選 重啟電腦 → 點擊 Specify an alternate source path

  7. 輸入 sxs Path

  8. 點擊 Install

設定 A Server 及 叢集

  1. Server Manager → Tools → 點擊 Network Load Balancing Manager

  2. 右鍵 → 點擊 New Cluster

  3. 輸入 A Server IP

  4. Next

  5. 點擊 Add

  6. 輸入 叢集 IP

  7. Next

  8. 選擇 Multicast

  9. Finish

設定 B Server

  1. 叢集右鍵 → Add Host To Cluster

  2. 輸入 B Server IP

  3. Priority 選 2

  4. Finish

  5. 設定完成

使用 Hexo 及 GitHub 建立 Blog

發表於 2019-04-26 | 分類於 Blog

順序

  1. 基本安基
  2. GitHub 帳號注冊及新增一個 Repository
  3. Hexo 建立
  4. Hexo + GitHub
  5. 部署

基本安基

安裝 Node

Node.js 下載並安裝

安裝 Git

Git for Windows 下載並安裝

安裝 Hexo

透過 npm 安裝 Hexo (出錯的地方，無視它)

GitHub 新增 Repository

  1. 開啟 github.com
  2. 建立一個 Repository
  3. 輸入自己的名字 username，後綴 .github.io

Hexo 建立

GitHub + Hexo

本機鏈結 GitHub

  1. 在新建的目錄按右鍵，開啟 git bash

  2. 生成 SSH 密鑰
    輸入完指令後，連按三次 Enter

  3. 密鑰加至 ssh-agent

  4. 生成的 SSH 加至 ssh-agent

  5. 開啟 github.com
    點擊頭像下 Settings

  6. 點擊左側 SSH and GPG keys
    點擊 New SSH key

  7. 顯示 SSH key 及複製

  8. 將 SSH key 貼至 GitHub SSH key 裡， Title 任意打

  9. 確認 GitHub 鏈結成功

Hexo config 修改

  1. 根目錄下開啟 _config.yml
    修改最後一行的配置，如下

    deploy:
     type: git
     repository: https://github.com/username/blogRepository.github.io
     branch: master

  2. 設定 Title、author、language
    修改 _config.yml

    title: your title
    author: your name
    language: zh-tw

  3. image 相對應路徑
    修改 _config.yml

    post_asset_folder: true

備份在另一個 Repository

  1. Commit Git

部署

  1. 清除快取檔案 (db.json) 和已產生的靜態檔案 (public)

  2. 產生靜態檔案

  3. 啟動伺服器

    預設是 http://localhost:4000/

  4. 部署網站

  5. Blog URL

    https://blogRepository.github.io/
